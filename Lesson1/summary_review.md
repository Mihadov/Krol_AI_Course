# Проверка резюме лекции

## Общий вывод

Резюме достаточно точно передаёт вводную, философскую и организационную часть лекции (разделы 1–6 резюме), а также базовый технический стек. Однако примерно половина документа (разделы 10–19, 22) — это **материал, которого в лекции нет**: подробные методологии работы с агентом, готовые шаблоны промптов, 30-дневная стратегия внедрения, чеклисты и FAQ в структурированном виде — всё это **додумано/достроено суммаризирующим агентом**, а не является содержанием первого урока. Конкретные технические рекомендации и инструменты, упомянутые лектором, в ряде мест пропущены или искажены.

## Что резюмировано точно ✅

- **Общий контекст курса**: лекция — стартовая точка, переход от ИИ как чата к ИИ как рабочей среде — передано верно.
- **Тезис об ИИ как лучшем учителе**: 4 причины (24/7, без осуждения, адаптация к уровню, бесконечные повторные объяснения) — в лекции присутствуют именно в таком духе.
- **Философия обучения**: «не искать идеальный курс», «сначала делать, потом понимать», метафора «квеста», где тупик — это дверь — всё это точно отражает содержание лекции.
- **Почему чат-режима недостаточно**: 4 причины перехода — ограничение контекстного окна, неопределённость ветвления, закрытость SaaS-среды, ограниченная операционность — в лекции перечислены явно и подробно, резюме передаёт их верно.
- **Базовый стек**: VS Code + Claude Code (от Anthropic) + Codex (от OpenAI) — назван правильно.
- **Важность форматов md/csv**: лектор демонстрирует конвертацию документов Pages/Numbers → plain text / Markdown / CSV и объясняет, почему Word/PDF плохо понимаются моделями — резюме это корректно отражает.
- **Принцип «не нужно быть разработчиком»**: стек одинаково применим к текстам, данным, автоматизации и коду — сказано прямо в лекции, в резюме есть.
- **Предупреждение насчёт расширений**: ставить только от официального издателя (Anthropic / OpenAI), не ставить клоны — в лекции это специально подчёркнуто, в резюме правильно отражено.
- **Нормализация «хаоса в голове» на старте** — упомянута в лекции многократно, в резюме есть.
- **Метафора «боссов в игре»** для каждого уровня — в лекции используется, в резюме отражена.
- **Различие Claude Code (парный режим / исполнитель) vs Codex (архитектор, автономные задачи)** — лектор явно это проговаривает, и резюме передаёт верно.
- **Ограничение памяти LLM как архитектурная проблема** — в лекции обсуждается подробно, резюме отражает правильно.

## Неточности и ошибки ❌

### 1. Codex — это не «расширение» в том же смысле, что Claude Code
**В резюме (8.2):** «Установить `Codex` от OpenAI» — как равнозначный агент в Extensions.  
**В лекции:** Лектор показывает как расширение VS Code, так и **отдельное десктопное приложение Codex** (которое «бесплатно цепляется к подписке ChatGPT Plus»). Разница существенна: десктопный Codex — самостоятельный инструмент, лектор демонстрирует его отдельно. Резюме про десктопную версию не говорит вообще.

### 2. Git/GitHub подаётся в резюме как часть базового стека первого урока
**В резюме (6.2):** «Git/GitHub — структура, история изменений, совместная работа».  
**В лекции:** Git/GitHub упоминается как **будущая тема** («мы будем говорить об этом позже», «когда начнём работать с GitHub»). В первом уроке он не показывается и не устанавливается. Включение его в «базовый стек первого урока» вводит в заблуждение.

### 3. Подписка: конкретные рекомендации лектора не отражены
**В резюме (12.4):** расплывчато: «серьёзная работа требует платных подписок».  
**В лекции:** лектор конкретен — **начинайте с Pro (Claude)**, Max не нужен на старте (100$/мес, только если упрётесь в лимиты); **ChatGPT Plus (20$/мес) достаточен** для работы с Codex на начальном уровне. Эти ориентиры важны для слушателей.

### 4. Описание разницы между агентами (раздел 9) — интерпретация, а не цитата
**В резюме (9.1–9.2):** Claude Code — «плотная парная работа», Codex — «более автономные задачи».  
**В лекции:** это верно, но лектор добавляет нюанс: разграничение «исполнитель / архитектор» уже устарело, потому что обе компании быстро «воруют фичи» друг у друга и разница нивелируется. Резюме представляет разграничение как устойчивое, лектор — как историческое.

### 5. Формула из раздела 22 отсутствует в лекции
**В резюме (22):** «Формула спикера: Идея → Задача → Структурированные данные → Агент в VS Code → Итерации → Проверка → Результат».  
**В лекции:** такой формулы нет. Это реконструкция агента, а не цитата или слова лектора.

## Важное, пропущенное в резюме ⚠️

### 1. Десятиуровневая классификация задач
Лектор неоднократно ссылается на собственный документ с **10-уровневой классификацией** AI-задач (от нулевого ликбеза до сложных агентных систем). Текущий курс покрывает уровни 3–8. Это структурный каркас, на котором строится курс — в резюме его нет.

### 2. Собственный open-source фреймворк лектора
Крол показывает свой **самодельный фреймворк** (репозиторий на GitHub: ~222 коммита, 30 форков, 177 звёзд на момент лекции), который решает проблему памяти между сессиями. Он работает с Claude Code и Codex. Фреймворк предлагается как опциональный инструмент — в резюме он почти не упомянут.

### 3. Конкретный инструмент конвертации PDF → Markdown
Лектор явно рекомендует **Gemini** для конвертации больших PDF в Markdown: «Gemini очень хорошо работает, он у меня справился». Это конкретная практическая рекомендация. Резюме говорит лишь «использовать ИИ-конверсию» без указания инструмента.

### 4. Сервисы для быстрого прототипирования (переходный уровень курса)
Лектор называет: **Bolt, Lovable, Vio, Netlify, Supabase** — как инструменты для простых лендингов и прототипов (переходный уровень курса). Даёт им оценку: удобны для быстрого фронта, но закрытая система, не подходят для сложных проектов. В резюме эти сервисы не названы.

### 5. Udemy как рекомендованный ресурс для самообразования
Лектор прямо называет **Udemy** («более-менее системная площадочка», ~80 млн уников в месяц, подписка ~$35/мес) и рекомендует его тем, кто хочет больше курсов. Резюме про Udemy не упоминает.

### 6. Десктопное приложение Claude с функцией CoWork
Лектор показывает десктопную версию Claude с функцией **CoWork**, называет её «просто бомба» и использует в рабочем процессе наравне с VS Code. В резюме Claude упомянут только как расширение VS Code.

### 7. Личные примеры применения стека (некодовые проекты)
Лектор показывает реальные примеры некодового использования стека:
- **Налоги**: утилита для конвертации P&L из QuickBooks в формат TaxAct (5 секунд написано с Claude Code).
- **Трекер 50+ проектов**: простое приложение, сделанное за ~20 минут с десктопным Codex.
- **Книга для издательства**: пишется в VS Code.
- **Художественное произведение**: командная работа в VS Code.
- **Замена GetResponse (~$450/мес)**: собственный mailer с Amazon SES, написанный с помощью ИИ.
Эти примеры — сильная мотивационная часть лекции, в резюме они практически отсутствуют.

### 8. Cursor и Antigravity — объяснение лектора
Лектор специально объясняет: **Cursor — это клон VS Code** (форк с добавленным кастомным кодом), **Antigravity — это тоже VS Code** (просто с Claude внутри). Поэтому можно использовать Antigravity и при этом добавить Claude Code как расширение — это одно и то же. Это важный ориентирующий момент для аудитории.

### 9. Почему лектор не рекомендует Bolt/Lovable при наличии Claude Code Max
Прямая рекомендация: «имеет смысл купить Bolt/Lovable, если есть Claude Code Max? Нет, не тратьте время». Резюме не даёт такой чёткой позиции.

### 10. Реальная инструкция по домашнему заданию
Лектор в конце прямо формулирует ДЗ: установить VS Code → установить Claude Code или Codex (хватит одного, рекомендует начать с Claude Code) → залогиниться → создать папку с файлами → перевести в md/csv → открыть в VS Code → начать диалог с агентом. Резюме (раздел 20) расширяет это до трёх блоков (A/B/C) с 12 шагами — структура оригинала проще.
